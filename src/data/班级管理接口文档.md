# 班级管理接口文档

Base URL: `/classManage`

所有接口均为 **POST** 请求，请求体为 JSON，响应体为 JSON。

---

## 通用说明

### 请求格式

```
Content-Type: application/json
```

### 身份认证

**所有接口均需在请求体中携带 `usr_id` 和 `token` 字段进行身份验证。**

token 为用户登录后获取的加密 token（RSA 加密），服务端通过 `encryptionTokenCheckIn` 方法解密并校验 token 与 usr_id 的绑定关系。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| usr_id | String | 是 | 当前操作用户的用户ID |
| token | String | 是 | RSA 加密的登录 token |

> `usr_id` 和 `token` 出现在每个接口的请求参数中，下文各接口不再重复说明这两个字段。

### 通用响应格式

```json
{
    "success": true/false,
    "message": "操作描述",
    "...": "其他业务数据"
}
```

### 错误响应

当请求参数解析失败或服务端异常时，HTTP 状态码返回 `400`，body 为字符串 `"err"`。

### Token 验证失败响应

```json
{
    "success": false,
    "message": "token_check_failed"
}
```

## 一、班级管理

### 1.1 创建班级

`POST /classManage/createClass`

**权限要求：** 操作者 VIP rank >= 10，创建成功后自动加入该班级。

**请求参数：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| usr_id | String | 是 | 操作者用户ID（教师） |
| token | String | 是 | 登录 token |
| class_name | String | 是 | 班级名称 |
| description | String | 否 | 班级描述 |

**请求示例：**

```json
{
    "usr_id": "teacher_001",
    "token": "加密token字符串",
    "class_name": "2026级计算机科学1班",
    "description": "计算机科学与技术专业"
}
```

**成功响应：**

```json
{
    "success": true,
    "class_id": "123456_1713500000000",
    "message": "Class created successfully"
}
```

**失败响应：**

```json
{
    "success": false,
    "message": "VIP rank must be >= 10 to create class"
}
```

```json
{
    "success": false,
    "message": "Only teachers can create classes"
}
```

### 4.1 添加学生到班级

`POST /classManage/addStudentToClass`

**权限要求：** 仅该班级的教师可操作（usr_id 对应的教师必须是该班级的教师）

**请求参数：**

| 字段 | 类型 | 必填 | 说明 |

| usr_id | String | 是 | 操作教师用户ID（必须是该班级的教师） |

```json
    "role": "member"
}
```

**成功响应：**


**失败响应：**

```json
{
```
